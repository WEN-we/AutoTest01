# 四种测试场景架构设计

## 场景定义

### 1. 测试平台本身（platform）
- **说明**：测试 web_platform 自身的功能
- **目标**：测试本平台的登录、Dashboard、任务管理等
- **配置项**：
  - 被测URL：http://localhost:8081
  - 无需额外认证

### 2. 测试外部网站（external）
- **说明**：测试外部互联网上的网站或系统
- **目标**：如教务系统、第三方API等
- **配置项**：
  - 被测URL：用户输入
  - 认证方式：可选（Token/Cookie/Basic Auth）
  - 请求头：自定义

### 3. 测试本地服务（local）
- **说明**：测试本地的服务，如 local_web_login
- **目标**：测试本地登录服务、API服务等
- **配置项**：
  - 服务地址：localhost
  - 端口：用户配置（如8080）
  - 数据库：本地MySQL

### 4. 其他（other）
- **说明**：自定义测试场景
- **目标**：特殊测试需求
- **配置项**：
  - 完全自定义
  - 支持多环境配置

## 技术实现

### 场景配置结构
```python
{
    "scene": "platform|external|local|other",
    "base_url": "被测系统地址",
    "port": "端口号",
    "auth": {
        "type": "token|cookie|basic|none",
        "token": "认证令牌"
    },
    "headers": {
        "自定义请求头": "值"
    }
}
```

### API修改
- 任务创建时增加 scene 字段
- 根据 scene 自动填充默认配置
- 执行时根据配置组装环境变量

### 前端交互
1. 选择场景后，显示对应的配置表单
2. 根据场景自动设置测试路径
3. 任务列表显示场景标签

## 1. 项目概述

本测试平台为企业级电商全平台自动化测试框架的Web管理界面，基于Python + Flask构建，提供完整的图形化操作界面。

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Flask (Python), MySQL, JWT认证 |
| 前端 | Bootstrap 5, HTML5, Vanilla JavaScript |
| 集成 | Jenkins, 禅道, AI模型 |

### 核心功能

- ✅ 用户认证与权限管理
- ✅ 测试任务配置与执行
- ✅ AI模型配置与管理
- ✅ 测试报告查看
- ✅ Jenkins集成（触发构建、查看状态）
- ✅ 禅道集成（用例同步、Bug管理）
- ✅ 响应式Bootstrap界面
- ✅ 日间/夜间主题切换

## 2. 快速开始

### 2.1 环境要求

- Python 3.8+
- MySQL 5.7+

### 2.2 安装依赖

```bash
cd d:\Pthon.Object\PythonProject3\web_platform
pip install -r requirements.txt
```

### 2.3 配置数据库

编辑配置文件 `backend/config/database.yaml`：

```yaml
development:
  host: "localhost"
  port: 3306
  database: "test_platform"
  username: "root"
  password: "${DB_PASSWORD}"
```

### 2.4 初始化数据库

```bash
cd backend
python scripts/init_database.py
```

这将创建：
- ✅ 新数据库 `test_platform`
- ✅ 所有数据表（6个表）
- ✅ 默认管理员账户

### 2.5 启动服务

**启动后端：**
```bash
cd backend
python app.py
```
后端运行在：**http://127.0.0.1:8081**

**启动前端：**
```bash
cd frontend
python -m http.server 8082
```
前端访问：**http://localhost:8082**

### 2.6 默认账户

- **用户名**: `admin`
- **密码**: `change_me_in_production`

**目标**：特殊测试需求

## 5. 集成配置

### 5.1 Jenkins集成

**功能**：
- 查看Jenkins任务列表
- 触发构建任务
- 查看构建历史
- 监控构建状态

**配置步骤**：
1. 进入"集成配置"页面
2. 点击"添加配置"
3. 选择集成类型：Jenkins
4. 填写配置信息：
   - 名称：Jenkins
   - 服务地址：http://jenkins-server:8080
   - 用户名：jenkins用户
   - API Token：jenkins的API Token

### 5.2 禅道集成

**功能**：
- 查看产品列表
- 查看测试用例
- 执行测试用例
- 创建Bug
- 同步测试用例

**配置步骤**：
1. 进入"集成配置"页面
2. 点击"添加配置"
3. 选择集成类型：禅道
4. 填写配置信息：
   - 名称：禅道
   - 服务地址：http://zentao-server/zentao
   - 账户：禅道账户
   - 密码：禅道密码

## 6. API接口

| 方法 | 路径 | 说明 |
|------|------|------|

## 7. 常见问题

### 7.1 数据库连接失败

**检查MySQL服务是否启动**
```bash
net start mysql
```

**检查配置是否正确**
- 编辑 `backend/config/database.yaml`
- 确认用户名密码正确

### 7.2 前端无法访问后端

**跨域问题**：后端已配置CORS，允许所有来源访问。

**检查端口**
- 后端运行在：8081
- 前端默认访问：8082
- 确保端口不冲突

### 7.3 Jenkins连接失败

1. 确认Jenkins地址可访问
2. 确认用户名和API Token正确
3. API Token获取：在Jenkins中 → 用户 → 配置 → API Token

### 7.4 禅道连接失败

1. 确认禅道地址可访问
2. 确认账户密码正确
3. 确认API权限开启
